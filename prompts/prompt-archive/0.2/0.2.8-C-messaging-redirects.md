# Prompt Archive: 0.2.8-C — Cross-Surface Redirects & Messaging UX

## Original Task
Plan cross-surface redirects and error/success messaging UX for auth/settings.

## Restatement
Define how auth/login/signup/reset/settings flows handle redirects between surfaces (e.g., post-login to dashboard, post-reset to login, settings save confirmation) and specify error/success messaging patterns that are i18n-ready, non-revealing, and accessible.

## Phase/Sub-phase
0.2.8-C

## Plan
- Map redirects for auth flows (login → dashboard, signup → welcome/dashboard, reset → login, logout → login) and settings save flows.
- Specify messaging UX: success toasts/banners, inline errors, non-revealing auth errors, focus management on errors/success.
- Ensure all messages are externalised for i18n, follow a11y patterns (aria-live), and use shared UI kit components.

## Outcomes
- Documented redirect map and messaging patterns for auth/settings with i18n/a11y requirements and shared UI kit usage; ready for implementation.

## Redirect Map (planned)
- Login success → dashboard (or intended URL if guard redirect param is present); ensure locale carried.
- Signup success → welcome/dashboard; if email verification later, route to verification screen.
- Reset request success → confirmation state on reset screen; optional link to login.
- Reset confirm (when added) → login with success banner.
- Logout → login with neutral message.
- Settings save → stay on settings; show success toast/banner; keep focus management.

## Messaging UX (planned)
- Success: toast or inline banner using shared UI kit; aria-live="polite"; externalised strings. Examples: “Logged in”, “Account created”, “Password reset link sent”, “Settings saved”.
- Errors: inline, non-revealing for auth (“Email or password is incorrect”); aria-live="assertive"; focus the first error field/message.
- Validation: per-field errors tied via `aria-describedby`; prevent double-submit; keep form state.
- Loading/disabled: disable submit while sending; show spinner text via i18n.
- A11y/i18n: all messages in shared i18n catalog; keep keyboard/focus order; avoid color-only cues.***
