# Prompt Archive: 0.2.12-B — Session Cookies & CSRF Protection

## Original Task
Use HTTP-only cookies for sessions/tokens; add CSRF protection for cookie flows; avoid localStorage.

## Restatement
Implement secure session handling with HTTP-only cookies, add a CSRF token/check for form/API requests that use cookies, and ensure no auth tokens are stored in localStorage. Update docs and client wiring accordingly.

## Phase/Sub-phase
0.2.12-B

## Clarifications
None yet.

## Plan
- Issue short-lived, HTTP-only, SameSite=Lax cookies for sessions; clear on logout/expiry.
- Add CSRF token issuance and validation for cookie-authenticated POST/PUT/PATCH/DELETE; include token in requests via header or hidden field.
- Audit client to avoid localStorage/sessionStorage for auth; rely on cookies + CSRF token.
- Update auth docs/changelog/prompt archive to reflect session/CSRF approach.

## Outcomes
- Session cookies enforced, CSRF protection added for cookie-based requests, no localStorage token usage; documentation and prompt archive updated.***
