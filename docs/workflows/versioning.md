# Versioning

## Purpose
Define how versions/tags are assigned for releases.

## Policy (initial)
- Use semantic versioning when packages/apps are published.
- Tag releases on main for production deployments.
- Keep changelog entries aligned with tags.

## Notes
- Revisit when publishing packages or apps; consider changeset tooling if needed.
- Coordinate with release process; ensure docs reflect tagged versions when applicable.***
- Compliance note: when tagging a release, confirm the changelog and docs capture consent/DSR implications and that tagged versions link to the corresponding standards updates.
