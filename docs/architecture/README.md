# Architecture Docs

This directory contains the platform’s architectural guidance. Follow the development outline and documentation hierarchy before implementing changes.

## Key Files
- `overview.md` — high-level system description and goals.
- `development-outline.md` — phased roadmap with sub-phases and tasks.
- `monorepo-structure.md` — required repo layout and package/apps.
- `services.md` — service inventory and responsibilities.
- `request-flow.md` — high-level request/response flow.
- `event-pipeline.md` — initial event flow for analytics/progress.
- `tooling.md` — stack, versions, and commands.
- `deployment-architecture.md` — deployment topology (to be expanded).
- `entity-system.md` / `permission-system.md` — models and RBAC (to be expanded).
